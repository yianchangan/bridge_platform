# 桥梁施工方案 Word 文档标准化交底

## 一、标题样式设置

### 1.1 必须使用 Word 内置标题样式

不要手工加粗、改字号来模拟标题。请用 Word 样式库中的**"标题 1"（Heading 1）到"标题 5"（Heading 5）**。

> 操作方法：选中标题文字 → 开始选项卡 → 样式区域 → 点击对应级别的标题样式。

### 1.2 确定标题层级

整理文档前，先确认原文最大到第几级标题（通常 3~5 级），然后上传至系统时填写 `max_heading_level`。

| 层级 | Word 样式名 | 示例 |
|------|------------|------|
| 第 1 级 | 标题 1 / Heading 1 | 第一章 工程概况 |
| 第 2 级 | 标题 2 / Heading 2 | 1.1 项目概况 |
| 第 3 级 | 标题 3 / Heading 3 | 1.1.1 工程简介 |
| 第 4 级 | 标题 4 / Heading 4 | （如有） |
| 第 5 级 | 标题 5 / Heading 5 | （如有） |

**注意**：
- 系统默认解析 1~5 级标题。如果文档实际只有 3 级标题，上传时将 `max_heading_level` 设为 3，避免正文中偶然出现的 Heading 4/5 被误拆。
- 如果文档的标题不是 Word 内置样式（比如用了自定义样式"一级"、"二级"等），需要联系管理员配置正则匹配规则。

## 二、图标题与表标题

### 2.1 样式命名

| 元素 | Word 样式名 | 系统默认匹配 |
|------|------------|-------------|
| 图标题 | 图标题 | ✅ |
| 表标题 | 表标题 | ✅ |

> 在 Word 中确保图标题和表标题段落应用了对应的样式。如果文档使用的是其他样式名（如"图片标题"、"表格名称"），上传时需在 `image_caption_style` 和 `table_caption_style` 参数中填入实际样式名。

### 2.2 文档中图片与图标题的对应关系

系统按照 **"图片 → 图标题"的顺序** 自动配对：

```
[图片实体]
图 2.1-1 项目地理位置示意图      ← 紧跟图片的图标题段落
```

**每张图片必须有独立的图标题**。如果原文中一个图标题下包含多张图片（如 a、b、c 子图），需拆分为多个图标题：

> **错误做法：**
> ```
> [图片 a]
> [图片 b]
> [图片 c]
> 图 4.5-2 立柱吊装示意图         ← 一个标题对应三张图，只能绑到 c
> ```

> **正确做法：**
> ```
> [图片 a]
> 图 4.5-2 立柱吊装示意图-1
> [图片 b]
> 图 4.5-2 立柱吊装示意图-2
> [图片 c]
> 图 4.5-2 立柱吊装示意图-3
> ```

表标题同理，多表同标题时在后缀加 `-1`、`-2`。

### 2.3 表格的放置顺序

系统按照 **"表标题 → 表格"的顺序** 自动绑定：

```
表 2.2-1 预制 T 梁立柱盖梁数量表   ← 表标题在前
┌────┬──────┬──────┐
│ 序号 │ 名称  │ 数量  │             ← 表格紧跟其后
├────┼──────┼──────┤
│    │      │      │
└────┴──────┴──────┘
```

如果表标题在表格之后，系统无法绑定，该表格不会被提取。

## 三、正文样式

### 3.1 确认正文使用的样式

文档中的正文不一定只有 `Normal`（正文）一种样式，可能还存在 `List Paragraph`（列表段落）、`Quote`（引用）、`List Bullet`（项目符号）等。

**操作流程：**

1. 用浏览器打开后端 API 文档页面：`http://<服务器地址>:8000/docs`
2. 调用 **`POST /api/documents/upload`** 上传 Word 文件（只需填文件 + 桥型 + 方案类型 + 名称）
3. 上传成功后，响应的 `scanned_styles` 字段会列出文档中所有段落样式及出现次数，例如：

```json
{
  "file": "施工方案.docx",
  "total_paragraphs": 853,
  "styles": {
    "Normal": 455,
    "Heading 1": 10,
    "Heading 2": 42,
    "Heading 3": 83,
    "Heading 4": 25,
    "图标题": 70,
    "表标题": 14,
    "List Bullet": 8,
    "Quote": 3
  }
}
```

4. 也可以单独调用 **`GET /api/documents/{doc_id}/styles`** 重新查看。

### 3.2 配置正文样式

查看样式分布后，将标题和图/表标题以外的**所有应纳入正文的样式**填入 `body_styles` 参数。多个样式用英文逗号分隔。

**示例：**

```
body_styles = "Normal,List Bullet,Quote"
```

> 默认值为 `Normal`。如果不填，只有 `Normal` 样式的段落会被当作正文入库。其他样式（如 `List Bullet`）的段落会被静默丢弃。

### 3.3 启动解析

确认样式配置无误后，调用 **`POST /api/documents/{doc_id}/parse`** 启动解析。需要填写的完整参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `max_heading_level` | 最大标题层级 | 5 |
| `body_styles` | 正文样式（逗号分隔） | Normal |
| `image_caption_style` | 图标题样式名 | 图标题 |
| `table_caption_style` | 表标题样式名 | 表标题 |

其他参数（`regex_level1`~`regex_level5`）为高级选项，一般情况下留空即可。

解析完成后调用 **`GET /api/documents/{doc_id}/status`** 确认进度为 100%、状态为 `PENDING_REVIEW`，再调用 **`GET /api/documents/{doc_id}/parsed`** 获取解析结果。

## 四、Word 整理自检清单

整理完成后，逐项检查：

- [ ] 所有标题使用了 Word 内置"标题 1"~"标题 N"样式，无手工加粗模拟
- [ ] 标题层级连续，不跳级（如无 Heading 3 直接到 Heading 4）
- [ ] 每个图标题只对应一张图，多图拆分为 `-1`、`-2` 后缀
- [ ] 表标题在表格之前
- [ ] 无空白段落使用标题样式
- [ ] 页眉页脚、目录、封面等内容不含需要入库的正文
